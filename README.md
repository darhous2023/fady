# ELFADY — معرض الفادي لتجارة السيارات

موقع تجاري حي وحقيقي لمعرض سيارات فعلي في المهندسين، القاهرة — سيارات جديدة ومستعملة، حجز معاينة، وبحث/مقارنة/مفضلة على كتالوج سيارات حقيقي، مبني بالكامل على Next.js وقاعدتَي بيانات منفصلتين، وتُدار كل صفحة فيه من لوحة أدمن واحدة بدون لمس كود.

**🔗 الموقع اللايف:** [fady-delta.vercel.app](https://fady-delta.vercel.app)
**📦 الريبو:** [github.com/darhous2023/fady](https://github.com/darhous2023/fady)
**📖 دليل الأدمن الحي:** [fady-delta.vercel.app/admin/guide](https://fady-delta.vercel.app/admin/guide)

> بيئة القبول النهائية هي **Vercel Production**. تشغيل الموقع محليًا لا يعني اكتمال العمل — دايمًا تحقق من الرابط اللايف بعد أي تغيير.

## ماذا يقدّم الموقع فعليًا

### للزائر / العميل
- **سيارات مستعملة** (`/used`) — سيارات حقيقية موجودة فعليًا في المعرض: صور متعددة، عارض 360° (سحب بالماوس/اللمس)، مواصفات كاملة (الماركة، الموديل، السنة، العداد، ناقل الحركة، الألوان، عدد الأبواب/المقاعد، المزايا...)، وتقييمات عملاء حقيقية تحت كل سيارة.
- **بوابة سيارات جديدة** (`/new`) — كتالوج حقيقي (**392 ماركة، 1200+ سيارة**) من قاعدة بيانات مستقلة، مع **بحث** (بما فيه تطابق أسماء عربية للماركات مثل "بي ام دبليو" → BMW)، **مقارنة** بين حتى 4 سيارات، و**مفضلة** — الاثنين الأخيرين محفوظين محليًا في المتصفح بدون تسجيل دخول.
- **شريط تمويل وتقسيط** متحرك في الصفحة الرئيسية، وبانرات ترويجية، وتقييمات عامة عن المعرض.
- **حجز معاينة** بدل "سلة تسوق" تقليدية — البيع الفعلي يحصل شخصيًا في المعرض، فالموقع يحجز ميعاد معاينة فقط (اسم، تليفون، تاريخ مفضّل، الفرع).
- **تتبّع الحجز** (`/track`) بمجرد كتابة رقم التليفون — يعرض كل حجوزات هذا الرقم بحالة كل واحد.
- تواصل مباشر عبر واتساب من أي صفحة، وكل بيانات التواصل (فيسبوك، إنستجرام، الخريطة، العنوان) قادمة من قاعدة البيانات لا من الكود.

### للأدمن (`/admin`)
لوحة واحدة تتحكم في كل شيء بدون كود: محتوى الصفحة الرئيسية (فيديو الهيرو، البوابتين، الشريط المتحرك، ركائز الثقة)، السيارات المستعملة (صور، 360°، مواصفات)، كتالوج السيارات الجديدة (إخفاء/إظهار/تعديل اسم عربي/ملاحظات/إضافة سيارات يدويًا)، الحجوزات (تشيك-ليست حالة الحجز)، التقييمات، البانرات، التمويل والتقسيط، الخصومات، الأقسام (الماركات)، العملاء، الصلاحيات، والإعدادات العامة (واتساب، السوشيال، العنوان). داشبورد بإحصائيات زوار حقيقية (من نظام تتبّع داخلي، بدون أي أداة تحليلات خارجية).

## البنية التقنية

- **Next.js 16** (App Router, Turbopack) + **React 19** + **TypeScript**
- **قاعدتا بيانات منفصلتان تمامًا** (عمدًا، بدون أي مشاركة اتصال أو schema):
  - **Supabase Postgres** — بيانات المتجر نفسه: السيارات المستعملة، الحجوزات، الأدمن، الإعدادات، التقييمات، البانرات، التمويل، تتبّع الزوار. RLS مفعّل على كل جدول حساس؛ الباك إند يتصل بصلاحية `BYPASSRLS`.
  - **Neon Postgres** (مشروع مستقل تمامًا) — كتالوج السيارات الجديدة فقط (الماركات/الموديلات/الفئات/المواصفات)، متزامن من مصدر خارجي عبر أداة sync خاصة، مع أعمدة `admin_hidden`/`admin_overrides` تسمح للأدمن بالتعديل دون أن يمحوها التزامن التالي.
- **Drizzle ORM** فوق كلا القاعدتين (schema/migrations منفصلة تمامًا لكل واحدة).
- **Better Auth** للمصادقة (حسابات الأدمن والعملاء) — بدون كتابة أي مصادقة يدويًا.
- **Supabase Storage** لرفع كل الصور (سيارات، بانرات، شعارات تمويل).
- **Tailwind CSS v3** + inline styles للمكونات المخصصة، **framer-motion** لحركات السكرول والانتقالات.
- **Vitest** لاختبارات الوحدة، وسكريبتات فحص مسارات/تحمّل حقيقية (`scripts/route-health-check.ts`, `scripts/load-test.ts`).
- استضافة **Vercel** (production + preview + development)، بوصل قاعدة بيانات عبر transaction-mode pooler (لا session-mode — درس مُتعلَّم من حادثة إنتاج حقيقية موثّقة في `.ai/`).

## البداية السريعة

```bash
npm install
cp .env.example .env.local   # ثم املأ القيم الحقيقية — لا تُرفع هذا الملف أبدًا
npm run dev                  # تطوير محلي على http://localhost:3000

npm run build                # تحقق قبل أي commit
npm run typecheck
npm test                     # vitest

npm run db:generate          # بعد أي تعديل على schema المتجر
npm run db:migrate           # تطبيق migrations المتجر (يستخدم MIGRATION_DATABASE_URL)
npm run cars:db:generate     # نفس الشيء لكتالوج السيارات الجديدة (Neon)
npm run cars:db:migrate

npm run routes:check         # يفحص كل مسار حي (متجر + أدمن + API) بحثًا عن 500/400 غير متوقع
npx tsx scripts/load-test.ts # اختبار تحمّل قراءة-فقط متحكَّم فيه ضد أي رابط
```

## هيكل المستودع

```text
src/app/                     صفحات ومسارات Next.js (المتجر، الأدمن، الـ APIs)
src/components/store/        مكونات الواجهة الأمامية (Hero، عارض 360، الفوتر، شريط التمويل...)
src/components/store/cars/   مكونات بوابة السيارات الجديدة (بحث، مقارنة، مفضلة)
src/components/admin/        مكونات لوحة الإدارة
src/lib/db/drizzle/          اتصال + schema + سكريبتات seed لقاعدة بيانات المتجر (Supabase)
src/lib/db/carsCatalog/      schema كتالوج السيارات الجديدة (Neon) — منفصل تمامًا
src/lib/cars/                طبقة القراءة/الكتابة لكتالوج السيارات الجديدة + البحث العربي
drizzle/migrations/          migrations قاعدة بيانات المتجر (إضافية دائمًا، لا تُحذف)
drizzle/cars-migrations/     migrations كتالوج السيارات الجديدة (منفصلة تمامًا)
scripts/                     seed، فحص مسارات، اختبار تحمّل، وسكريبتات لمرة واحدة
.ai/                         ذاكرة المشروع الكاملة لجلسات Claude Code (git-ignored)
docs/                        وثائق موروثة من القالب الأصلي (عامة، ليست خاصة بـ elfady)
```

## ذاكرة المشروع لوكلاء الذكاء الاصطناعي

مجلد `.ai/` (غير مرفوع على git) هو مصدر الحقيقة الكامل والمُحدَّث لحظيًا لحالة المشروع: القواعد الثابتة (`CLAUDE.md`)، الحالة الحالية (`PROJECT_STATE.md`)، القرارات وأسبابها (`DECISIONS.md`)، ملخص سريع للاستئناف (`HANDOFF.md`)، والمهام المتبقية (`TODO.md`). أي جلسة عمل جديدة (بشرية أو وكيل ذكاء اصطناعي) يجب أن تبدأ بقراءة هذا المجلد قبل أي تعديل.

## الأمان

- RLS مفعّل على كل جدول حساس في قاعدة بيانات المتجر؛ لا سياسة "backend can manage" عامة — الباك إند يتجاوز RLS عبر صلاحية الاتصال نفسها لا عبر سياسة قابلة للاستغلال.
- لا كلمة مرور تُكتب أبدًا في نموذج تسجيل دخول الأدمن أثناء أي اختبار تلقائي — أي اختبار E2E يستخدم واجهة المصادقة الخلفية مباشرة (انظر `scripts/e2e-admin-session-*.ts`).
- لا سرّ (مفتاح Supabase، رمز Vercel، باسورد قاعدة بيانات) يُكتب في أي ملف مرفوع على git، بما في ذلك `.ai/`.

## الأسرار والبيئة

راجع `.env.example`. الأسرار الحقيقية موجودة في `.env.local` فقط (git-ignored)، ومضبوطة على Vercel لكل من production/preview/development.

## ملاحظة عن `docs/`

مجلد `docs/` والملفات الجذرية الأخرى (`AGENTS.md`, `PROMPT.md`, `MASTER_EXECUTION_SPEC.md`, إلخ) موروثة من `Store-Master-Template` الأصلي وتصف القالب كأداة عامة لإنشاء متاجر جديدة — وليست خاصة بمعرض الفادي تحديدًا. المرجع الفعلي والمحدَّث لحالة هذا المشروع تحديدًا هو `.ai/` كما هو موضّح أعلاه.
