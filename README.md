# ELFADY — معرض الفادي لتجارة السيارات

موقع ويب فعلي وحيّ لمعرض سيارات حقيقي في المهندسين، القاهرة — سيارات جديدة ومستعملة، مبني على بنية Next.js/Drizzle/Supabase مأخوذة من قالب متجر أزياء إلكتروني (`Store-Master-Template`)، مع واجهة أمامية وهوية بصرية مُعاد بناؤها بالكامل من الصفر لتناسب معرض سيارات.

**الموقع اللايف:** [fady-delta.vercel.app](https://fady-delta.vercel.app)
**الريبو:** [github.com/darhous2023/fady](https://github.com/darhous2023/fady)

> بيئة القبول النهائية هي Vercel production. تشغيل الموقع محليًا لا يعني اكتمال العمل — دايمًا تحقق من الرابط اللايف بعد أي تغيير.

## ماذا يقدّم الموقع؟

- **سيارات مستعملة** (`/used`) — سيارات حقيقية معروضة فعليًا في المعرض، بصور ومواصفات موثّقة (الماركة، الموديل، السنة، العداد، ناقل الحركة، درجة الحالة)، مُدارة بالكامل من لوحة الأدمن.
- **بوابة سيارات جديدة** (`/new`) — تصفّح حي لكل الماركات والموديلات عبر CarAPI، مع استعلام مباشر عن التوفر عبر واتساب.
- **عارض 360°** — سحب بالماوس أو اللمس لتدوير صور السيارة، مبني بدون أي مكتبات خارجية.
- **حجز معاينة** بدل "سلة تسوق" تقليدية — العميل يحجز ميعاد معاينة بدل شراء أونلاين، لأن البيع الفعلي يحصل شخصيًا في المعرض.
- **لوحة إدارة كاملة** (`/admin`) — كل محتوى الموقع (السيارات، الصفحة الرئيسية، الأقسام، الحجوزات، البانرات) قابل للتعديل من قاعدة البيانات بدون لمس الكود. راجع [/admin/guide](https://fady-delta.vercel.app/admin/guide) للدليل الكامل.

## البنية التقنية

- **Next.js 16** (App Router, Turbopack) + **React 19** + **TypeScript**
- **Drizzle ORM** فوق **Supabase Postgres** (استخدم اتصال الـ pooler، الاتصال المباشر IPv6-only وغير متاح من هذه البيئة)
- **Better Auth** للمصادقة (حسابات الأدمن والعملاء)
- **Supabase Storage** لرفع الصور
- **Tailwind CSS v3** + inline styles للمكونات المخصصة
- **framer-motion** لحركات السكرول والانتقالات

## البداية السريعة

```bash
npm install
cp .env.example .env.local   # ثم املأ القيم الحقيقية — لا تُرفع هذا الملف أبدًا
npm run dev                  # تطوير محلي
npm run build                # تحقق قبل أي commit
npm run db:generate          # بعد أي تعديل على schema
npm run db:migrate           # تطبيق migrations (يستخدم MIGRATION_DATABASE_URL)
```

## هيكل المستودع

```text
src/app/                 صفحات ومسارات Next.js (المتجر، الأدمن، الـ APIs)
src/components/store/    مكونات الواجهة الأمامية (Hero، عارض 360، الفوتر، إلخ)
src/components/admin/    مكونات لوحة الإدارة
src/lib/db/drizzle/      اتصال قاعدة البيانات، schema، سكريبتات seed
drizzle/migrations/      كل migrations قاعدة البيانات (إضافية دائمًا، لا تُحذف)
scripts/                 سكريبتات seed وإصلاح بيانات لمرة واحدة
.ai/                     ذاكرة المشروع لجلسات Claude Code (git-ignored)
docs/                    وثائق موروثة من القالب الأصلي (عامة، ليست خاصة بـ elfady)
```

## ذاكرة المشروع لوكلاء الذكاء الاصطناعي

مجلد `.ai/` (غير مرفوع على git) هو مصدر الحقيقة الكامل والمُحدَّث لحظيًا لحالة المشروع: القواعد الثابتة (`CLAUDE.md`)، الحالة الحالية (`PROJECT_STATE.md`)، القرارات وأسبابها (`DECISIONS.md`)، ملخص سريع للاستئناف (`HANDOFF.md`)، والمهام المتبقية (`TODO.md`). أي جلسة Claude Code جديدة يجب أن تبدأ بقراءة هذا المجلد.

## الأسرار والبيئة

راجع `.env.example`. لا تضع أي مفتاح Supabase، رمز Vercel، أو كلمة مرور في ملف مرفوع على git. الأسرار الحقيقية موجودة في `.env.local` فقط (git-ignored)، ومضبوطة على Vercel production/preview/development.

## ملاحظة عن `docs/`

مجلد `docs/` والملفات الجذرية الأخرى (`AGENTS.md`, `PROMPT.md`, `MASTER_EXECUTION_SPEC.md`, إلخ) موروثة من `Store-Master-Template` الأصلي وتصف القالب كأداة عامة لإنشاء متاجر جديدة — وليست خاصة بمعرض الفادي تحديدًا. المرجع الفعلي والمحدَّث لحالة هذا المشروع تحديدًا هو `.ai/` كما هو موضّح أعلاه.
