# تعليمات التشغيل

## 1. ما هو المشروع؟

هذا المشروع هو **Store Master Template**: نسخة أساس آمنة وموثقة من متجر ShahY Store، جاهزة ليستخدمها Claude أو Codex أو Gravity لإنشاء متجر جديد بدون أن تشرح المعمارية من الصفر.

المشروع يحافظ على واجهة المتجر، لوحة التحكم، قاعدة البيانات، المصادقة، التخزين، وأسلوب النشر على Vercel. أنت تتحكم في اسم المتجر، الهوية، المنتجات، الصلاحيات، الدومين، وحسابات GitHub/Supabase/Vercel.

## 2. أسرع طريقة لإنشاء متجر جديد

1. أنشئ مجلدًا جديدًا فارغًا.
2. أنشئ GitHub repository خاص.
3. افتح هذا القالب مع Claude أو Codex أو Gravity.
4. أعط الوكيل برومبت بدء متجر جديد.
5. أرسل هوية المتجر مرة واحدة.
6. سجّل دخول GitHub.
7. سجّل دخول Supabase.
8. سجّل دخول Vercel.
9. اترك الوكيل ينفذ حتى النشر.
10. افتح رابط Vercel الحي.
11. افتح `/admin/login`.
12. أضف أو عدّل المنتجات من لوحة التحكم.

## 3. البرومبت الجاهز لبدء متجر جديد

```text
اقرأ AGENTS.md وPROMPT.md وPROJECT_CONTEXT.md وملف تعليمات التشغيل.md بالكامل. ابدأ إنشاء متجر جديد من الـMaster Template. اسألني أولًا عن هوية المتجر في رسالة واحدة، ثم اطلب مني كل الصلاحيات الناقصة في رسالة واحدة، وبعد استلامها نفّذ المشروع كاملًا حتى النشر على Vercel واختبار النسخة الحية. لا تعتبر localhost نتيجة نهائية.
```

English:

```text
Read AGENTS.md, PROMPT.md, PROJECT_CONTEXT.md, STORE_IDENTITY_TEMPLATE.md, README.md, and docs/. Start a new store from this Master Template. Ask me once for store identity and once for missing credentials. Deploy to Vercel and verify the live URL. Do not treat localhost as completion.
```

## 4. البيانات التي سيطلبها الـAgent

مطلوب: اسم المتجر، نوع النشاط، اللوجو أو موافقة على placeholder، الاتجاه البصري.

اختياري: الألوان، الخطوط، الجمهور، العملة، اللغة، بيانات التواصل، السوشيال، الدومين، مواقع مرجعية. لو لم ترسلها سيستخدم الوكيل defaults آمنة.

## 5. صلاحيات GitHub

GitHub مطلوب لحفظ الكود:

```powershell
gh auth login
gh auth status
git remote -v
```

تأكد أن remote يشير لمستودع المتجر الجديد، وليس ShahY Store.

## 6. صلاحيات Supabase

Supabase مطلوب للمنتجات، الطلبات، العملاء، الصور، الإعدادات، وRLS. قد يطلب الوكيل `DATABASE_URL`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, و`SUPABASE_SERVICE_ROLE_KEY`. المفتاح service-role خاص جدًا ولا يوضع في المتصفح أو GitHub.

## 7. صلاحيات Vercel

```powershell
vercel login
vercel whoami
```

Vercel هو الإنتاج النهائي. يضيف الوكيل environment variables وينشر ويفحص الرابط الحي.

## 8. تشغيل المشروع محليًا

```powershell
npm install
npm run dev
npm run lint
npm run typecheck
npm run build
```

## 9. النشر على Vercel

نجاح localhost لا يعني اكتمال المتجر. النتيجة النهائية هي رابط Vercel الحي بعد اختباره.

## 10. استخدام لوحة التحكم

افتح `/admin/login`. بعد تسجيل الدخول يمكنك إدارة المنتجات، الأقسام، الطلبات، الشحن، الخصومات، البانرات، عروض الفلاش، العملاء، الإعدادات، والصلاحيات حسب `docs/ADMIN_CAPABILITY_MATRIX.md`.

## 11. أين توجد البيانات؟

المنتجات والأقسام والطلبات والبيانات الديناميكية في قاعدة البيانات. الأسرار في environment variables. الأصول البصرية لها مواقع موثقة. لا تستخدم arrays hardcoded كبديل للإنتاج.

## 12. تحديث الهوية فقط

```text
غيّر هوية المتجر فقط: الاسم، اللوجو، الألوان، الخطوط، الصور، tone، وmetadata. لا تغيّر وظائف التجارة، لوحة الأدمن، قاعدة البيانات، auth، RLS، أو طريقة النشر.
```

## 13. استكمال عمل متوقف

```text
اقرأ Git status وآخر commit وdocs/FINAL_HANDOFF.md وdocs/verification/latest-results.md. تحقق من Vercel live URL، ثم أكمل العمل بدون إعادة تنفيذ ما تم.
```

## 14. فحص متجر موجود

اطلب من الوكيل تدقيق GitHub، Supabase، Vercel، لوحة الأدمن، المتجر الحي، security، وdata ownership.

## 15. حل المشكلات

- Build failure: شغّل `npm run build`.
- Missing env: قارن `.env.local` مع `.env.example`.
- Supabase error: تحقق من project ref وDATABASE_URL.
- Images not loading: تحقق من bucket وpublic URL.
- Admin auth failure: تحقق من Better Auth و`/admin/login`.
- Products not appearing: تحقق من status=active والصور والـcategory.
- Wrong remote: شغّل `git remote -v`.
- Secrets detected: احذف الملف أو استبدل القيمة placeholder قبل commit.

## 16. النسخ الاحتياطي والاسترجاع

قبل migrations مدمرة أو تغييرات إنتاجية، خذ backup من Supabase وقم بتأكيد Git commit. لا تستخدم أوامر حذف أو reset بدون سبب واضح وموافقة.

- [ ] لوحة الأدمن تعمل.
- [ ] الصور تعمل.
- [ ] الموبايل والديسكتوب جيدان.
- [ ] SEO صحيح.
- [ ] Vercel live URL يعمل.
- [ ] الأسرار غير مرفوعة.
- [ ] GitHub يحتوي آخر commit.

## 18. ممنوعات مهمة

لا ترفع `.env`. لا تعرض service-role key. لا تعتبر localhost نتيجة نهائية. لا hardcode للمنتجات. لا تربط متجرًا جديدًا بإنتاج ShahY. لا تحذف لوحة الأدمن. لا تعيد بناء المشروع من الصفر بلا ضرورة مثبتة.
